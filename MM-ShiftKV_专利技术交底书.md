## 1、发明名称（3个候选）
1. 一种面向多模态大语言模型的解码感知预填充阶段键值缓存选择方法。
2. 一种用于多模态推理服务的低显存键值缓存压缩与稳定调度方法。
3. 一种神经网络自回归推理中的分布对齐式历史状态选择方法及系统。

## 2、要解决的技术问题
现有 prefill-only KV 选择方法通常默认“预填充阶段查询分布可代表解码阶段查询分布”。该假设在多模态场景下失效，导致基于预填充统计得到的 KV 重要性评估出现偏差。其后果为：关键视觉 token 被误删、严格预算下性能快速退化、端到端生成质量和时延稳定性下降。

## 3、最接近现有技术及缺点
### 3.1 窗口/启发式方法
通过保留最近窗口或固定规则 token 控制 KV 规模。

缺点：难识别跨位置关键视觉信息；低预算鲁棒性差；任务迁移稳定性不足。

### 3.2 预填充统计打分方法
基于预填充局部注意力、激活统计或 Key 相似度对 KV 打分并一次性裁剪。

缺点：未显式建模 prefill/decode 分布偏移；预算紧张时排序波动明显；关键 token 覆盖不稳定。

### 3.3 解码阶段动态重排/淘汰方法
在解码过程中持续重排或淘汰 KV。

缺点：引入在线计算开销和时延抖动；系统实现复杂；与高性能推理内核协同成本较高。

## 4、本发明技术方案
### 4.1 系统模块
1. 统计模块：提取 prefill 隐状态统计量 `μ_pre、σ_pre`。
2. 代理查询生成模块：执行方差扩展采样、查询投影与未来位置对齐。
3. 重要性评估模块：计算代理查询对 Prompt Keys 的注意力，进行分组聚合与投票。
4. 预算选择模块：按 `vote + anchor` 稳定排序并进行预算内选择。
5. 推理执行模块：prefill 一次执行，decode 阶段不干预 Prompt KV。

### 4.2 核心流程（S1-S8）
S1：对每层 prefill 隐状态计算按维均值 `μ_pre^(l)` 与标准差 `σ_pre^(l)`。  
S2：设置方差扩展因子 `γ>1`，采样 `N` 个代理隐藏状态。  
S3：经 `WQ^(l,h)` 投影得到代理查询 `q̃_i^(l,h)`。  
S4：对代理查询施加“未来位置”RoPE 以对齐解码查询分布。  
S5：计算代理查询对 Prompt Keys 的注意力分布。  
S6：将代理查询划分为 `G` 组，按阈值 `τ` 选组内关键 keys 并累积 `vote(t)`。  
S7：按 `score(t)=vote(t)+λ·anchor(t)` 排序，其中 `anchor` 仅用于平票稳定。  
S8：在预算 `C(l,h)` 下保留 `Top(C(l,h)-1)`，并强制保留最近 token。

### 4.3 数学表达与伪代码
关键公式：

\[
\mu_{pre}^{(l)}=\mathrm{mean}(H^{(l)}),\quad
\sigma_{pre}^{(l)}=\mathrm{std}(H^{(l)})
\]
\[
\tilde{H}_i^{(l)}\sim \mathcal{N}\big(\mu_{pre}^{(l)},\mathrm{diag}((\gamma\sigma_{pre}^{(l)})^2)\big)
\]
\[
\tilde{q}_i^{(l,h)} = W_Q^{(l,h)}\tilde{H}_i^{(l)}
\]
\[
a_t(\tilde{q}_i)=\mathrm{softmax}\left(\frac{\tilde{q}_i^\top k_t}{\sqrt{d}}\right)
\]
\[
\mathrm{score}(t)=\mathrm{vote}(t)+\lambda\cdot \mathrm{anchor}(t)
\]

伪代码：

```text
Input: Prompt KV, C(l,h), γ, N, G, τ, λ
Output: Compressed Prompt KV

for each layer l:
  compute μ_pre(l), σ_pre(l)
  sample N proxy hidden states with expanded variance
  for each head h:
    project proxies with WQ(l,h)
    apply future-position RoPE
    compute attention to all prompt keys
    group proxies into G groups
    aggregate attention mass and vote keys by τ
    score = vote + λ * anchor
    keep recent key + top(C(l,h)-1) keys
return compressed KV
```

### 4.4 约束与复杂度
1. 预算约束：`|C'(l,h)| ≤ C(l,h)`。
2. 时间复杂度：每层每头主项约为 `O(N·|T|)`。
3. 空间复杂度：流式实现可将新增峰值内存控制在 `O(|T|)`。
4. 兼容性：仅改 prefill 末端选择逻辑，不改 decode 主路径；兼容 FlashAttention 与 GQA/MQA。

### 4.5 实施例
实施例 A（标准图文问答场景）  
输入条件：图像+问题文本。  
参数设置：中等预算（如每头 256），采用默认 `γ、τ、N、G、λ`。  
执行步骤：按 S1-S8 执行后进入正常 decode。  
输出效果：在保持精度前提下降低显存占用，并改善时延。

实施例 B（长上下文文档/OCR 场景）  
输入条件：文档图像+长文本提示，序列长度较大。  
参数设置：在低预算（如每头 64/128）下，适度增大 `N` 或 `G`。  
执行步骤：prefill 一次性选择，decode 不重排 Prompt KV。  
输出效果：低预算下关键字符区域 token 保留更稳定，端到端质量波动降低。

## 5、技术效果（与第3部分缺点对应）
1. 针对分布失配：通过方差扩展+未来位置对齐，提高解码相关性建模能力。
2. 针对低预算退化：通过分组投票抑制评分噪声，提升排序稳定性。
3. 针对解码时延抖动：仅 prefill 执行，decode 阶段零干预。
4. 针对部署复杂度高：不改主推理链路，工程集成成本较低。
5. 针对精度/显存/时延难平衡：在同等预算下提升关键 token 覆盖率，从机制上改善低预算稳定性。

## 6、可替代方案
1. 代理采样由高斯采样替换为分位数重采样或混合分布采样。
2. 分组投票替换为分位数聚合、稳健均值或温度加权聚合。
3. `anchor` 由末查询注意力替换为最近 `k` 个真实查询均值。
4. `C(l,h)` 由固定预算替换为层重要性自适应预算。

## 7、关键创新点提炼（用于权利要求）
- 特征 F1：在 prefill 阶段构造解码感知代理查询用于 KV 选择。
- 特征 F2：使用方差扩展因子 `γ` 校准查询分布尺度。
- 特征 F3：代理隐藏状态经 `WQ` 映射到层-头查询空间。
- 特征 F4：代理查询施加未来位置 RoPE 进行位置分布对齐。
- 特征 F5：基于注意力质量进行分组聚合并投票评分。
- 特征 F6：通过阈值 `τ` 形成组内最小覆盖 key 集合。
- 特征 F7：采用 `vote + λ·anchor` 的稳定排序机制。
- 特征 F8：预算内 Top 选择并强制保留最近 token。
- 特征 F9：全流程仅 prefill 一次执行，decode 不重排 Prompt KV。
- 特征 F10：兼容 FlashAttention 与 GQA/MQA 结构。

## 8、术语解释
1. Prefill：对输入提示序列一次性前向计算并建立 Prompt KV 的阶段。
2. Decode：自回归逐 token 生成阶段。
3. Prompt KV：由提示输入生成、解码阶段反复访问的历史 KV。
4. RoPE：旋转位置编码机制，用于引入相对位置信息。
5. GQA/MQA：分组查询注意力/多查询注意力结构。
6. Attention Mass Coverage：被保留 Prompt KV 承载的解码注意力质量占比。

## 10、审查风险与规避建议
### 10.1 新颖性风险
风险点：可能被认为与现有 prefill 打分+TopK 方案相近。  
规避：强调“方差扩展+未来位置对齐+分组质量投票+平票锚定+仅 prefill 执行”的组合技术特征。

### 10.2 创造性风险
风险点：可能被质疑为常规参数调优。  
规避：补充低预算场景下的稳定性证据，证明组合机制的协同效果不可由单一参数替代。

### 10.3 公开充分风险
风险点：参数边界、异常处理与工程细节披露不足。  
规避：补充参数推荐区间、边界条件处理、复杂度与部署接口说明。
