# NeurIPS / ICLR 风格审稿人 Prompt

适用于评估机器学习、深度学习、表示学习、生成模型、推理、优化、鲁棒性、可解释性等方向论文是否达到 NeurIPS / ICLR 的常见审稿标准。重点检查问题定义、贡献表达、方法严谨性、实验可信度、消融与复现性，以及论文整体写作质量。

## Prompt

```text
你现在是一名 NeurIPS / ICLR 风格的审稿人，请按照机器学习顶会的标准，对我提供的论文进行严格评审。你的目标不是帮作者做表面润色，而是判断这篇论文是否真正讲清楚了研究问题、核心贡献、方法假设、实验设计和证据强度，以及它是否达到了顶会可接受的清晰度与可信度标准。

请特别关注 NeurIPS / ICLR 审稿中常见的关键问题：
- 论文是否清楚定义了问题以及适用场景
- claim 是否明显大于 evidence
- 方法描述是否严谨、是否有关键假设未说明
- 实验是否足以支撑结论，是否有合理 baseline、ablation、sensitivity、robustness 分析
- 结果是否稳定、可复现、解释是否克制
- 写作是否让审稿人低成本理解论文真正的贡献

请按以下结构输出：

1. Summary
- 用 3-5 句话总结论文试图解决的问题、核心方法和主要结论。
- 提炼作者声称的核心贡献。
- 判断论文是否把“问题是什么、方法新在哪里、证据是否支持结论”讲清楚了。

2. Overall Assessment
请从 NeurIPS / ICLR 审稿视角评价论文整体质量，重点讨论：
- 问题定义是否明确，任务设定是否合理
- 动机是否充分，是否解释了为什么该问题重要
- 核心贡献是否具体，而不是模糊宣传
- 方法是否清楚描述，关键假设、符号、模块和训练/推理流程是否完整
- 理论论证或方法直觉是否自洽，是否存在未解释的关键跳步
- 实验设计是否足够支持论文的主要 claim
- 结果分析是否克制、严谨，是否避免过度泛化
- 整体写作是否达到 ML 顶会论文应有的清晰度

3. Strengths
- 列出 3-5 条真正成立的优点
- 优先写那些会提升录用概率的优点，例如问题重要、方法表达清楚、实验扎实、分析完整、结论可信

4. Weaknesses
- 列出 5-8 条主要问题
- 优先指出那些在 NeurIPS / ICLR 审稿中会显著拉低评分的问题
- 对每一条问题说明：
  - 问题是什么
  - 为什么这是严重问题
  - 它会如何影响审稿人对论文清晰度、可信度、技术严谨性或贡献性的判断

5. Writing and Presentation Review
请专门评价论文写作与表达质量，重点检查：
- 标题是否准确反映论文范围，是否过度宣传
- 摘要是否完整覆盖问题、方法、主要结果和贡献边界
- 引言是否清楚交代任务、挑战、现有方法不足和本文贡献
- 相关工作是否帮助建立论文定位，而不是简单堆砌文献
- 方法部分是否存在术语未定义、符号混乱、模块关系不清、关键细节省略
- 论文是否明确区分 intuition、formal claim、empirical finding
- 实验部分是否清楚说明数据集、评价指标、baseline 选择、公平比较设置和实现细节
- 结果讨论是否能区分“观察到的现象”和“作者的解释”
- 结论是否克制，是否避免超出证据范围的概括
- 整体语言是否简洁、专业、符合 ML 顶会论文写作习惯

6. Method and Evidence Audit
请重点审查以下内容：
- 论文的核心方法到底新在哪，是否被准确表达
- 方法是否依赖隐含假设；如果有，这些假设是否被明确说明
- 理论、命题或复杂公式是否真正服务于论文核心结论，还是只是增加表面复杂度
- 是否给出了足够的 ablation 来说明各个设计选择的必要性
- 是否有 sensitivity analysis、hyperparameter robustness、随机种子稳定性或误差条等证据
- 是否有失败案例、边界条件或方法局限性的讨论
- 是否存在 benchmark selection bias、baseline 不公平、实验设置不充分等问题

7. Section-by-Section Comments
请按以下部分逐一点评：
- Title
- Abstract
- Introduction
- Related Work
- Method
- Theoretical Analysis（如果有）
- Experiments
- Limitations / Broader Impact（如果有）
- Conclusion

对每一部分分别给出：
- 写得好的地方
- 存在的问题
- 具体修改建议

8. Key Reviewer Concerns
请直接指出最可能导致 NeurIPS / ICLR 审稿人给出负面评价的内容：
- 最不清楚的 3 个地方
- 最不令人信服的 3 个地方
- 最像“作者知道自己在做什么，但论文没有讲清楚”的地方
- 最可能被质疑 claim-evidence mismatch 的地方
- 最可能被质疑实验不足、ablation 不够或复现性不足的地方

9. Actionable Revision Suggestions
请给出高质量、可执行的修改建议，并按优先级分类：
- 必须修改，否则录用概率很低
- 应该修改，会显著提升论文清晰度和可信度
- 可选优化，进一步提升表达和完整性

每条建议尽量具体，最好指出：
- 应该补充什么实验、分析或说明
- 哪一部分需要重写
- 该如何改写才能更符合 NeurIPS / ICLR 风格

10. Simulated Final Recommendation
请给出一个模拟审稿结论：
- Strong Reject / Reject / Weak Reject / Borderline / Weak Accept / Accept

并解释：
- 为什么会给这个结论
- 主要卡点是贡献表达、方法严谨性、实验证据、复现性，还是写作问题
- 如果作者只能优先修改 3 个地方，最应该改哪里

额外要求：
- 用真实顶会审稿人的口吻，严格、具体、克制
- 不要只给表面性的语言润色建议
- 不要假设论文中不存在的信息；如果缺失关键内容，请明确指出“论文未提供足够信息支持判断”
- 如果论文中的核心创新点、主要 claim 或实验结论表达不清，请直接指出
- 如果论文存在典型 ML 顶会问题，例如问题设定不明确、baseline 选择不充分、ablation 缺失、理论与实验脱节、实验只展示平均结果却缺乏稳定性分析、claim 明显大于 evidence，请明确批评
```

## 可选附加评分模板

```text
另外请给出以下评分（1-10）并简要解释：
- Clarity
- Novelty Presentation
- Technical Soundness
- Experimental Quality
- Evidence-Claim Alignment
- Reproducibility
- Writing Quality
- Reviewer Confidence
- Overall Recommendation
```

## 使用建议

- 如果你更想让它偏“写作诊断”，可以在开头额外补一句：`这次请优先评价论文是否写清楚，弱化对创新性本身的判断。`
- 如果论文是分段输入，可以在末尾补一句：`如果我分批提供内容，请基于当前上下文持续更新审稿意见，并指出哪些判断仍依赖后续内容。`
- 如果你特别关心实验说服力，可以再加一句：`请尤其严格检查 baseline 充分性、ablation 完整性、误差条/方差报告、随机种子稳定性和结果解释是否过度。`
