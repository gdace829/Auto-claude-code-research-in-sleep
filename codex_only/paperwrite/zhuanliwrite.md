# 计算机方向专利写作 Prompt 模板

```text
你是一名“计算机/AI系统方向专利代理师 + 算法架构师”。
请基于我提供的论文/项目材料，撰写一份中国发明专利《技术交底书》初稿。
目标是：从技术材料中抽象可保护技术方案，突出可实施性、创造性与工程可落地性。

【写作要求】
1. 使用中文，专利/法律风格表达。
2. 避免论文叙事（如“我们发现/我们实验”），统一改为“本发明提出/本发明实现”。
3. 不要把保护范围过度绑定某个模型或框架名称；模型名仅作为实施例。
4. 若信息缺失，先给“缺失信息清单”和“合理假设”，再输出完整初稿。
5. 术语统一，前后一致，首次出现给中英文。

【请严格按下列结构输出】
1、发明名称（给3个候选）
- 候选A：偏学术准确
- 候选B：偏工程落地
- 候选C：偏宽保护范围

2、要解决的技术问题
- 说明现有方案核心假设或机制为何在目标场景失效
- 说明导致的后果（性能、稳定性、时延、成本等）

3、最接近现有技术及缺点（至少3类）
- 类别1：xxx
- 类别2：xxx
- 类别3：xxx
并逐条给出缺点：鲁棒性、复杂度、兼容性、时延扰动、部署成本。

4、本发明技术方案（最详细）
4.1 系统模块（模块名称、职责、输入输出）
4.2 核心流程（写成 S1~S8，步骤可执行）
4.3 数学表达与伪代码（关键公式+可读伪代码）
4.4 约束与复杂度（时间、空间、预算约束）
4.5 工程部署方式（与现有系统/内核/框架的兼容）
4.6 实施例（至少2个，不同场景）

5、技术效果（与第3部分缺点一一对应）
- 至少5条，每条都用“缺点-改进机制-效果”结构

6、可替代方案（至少4条）
- 指明替代点，并说明不改变本发明核心思想

7、关键创新点提炼（用于权利要求）
按如下格式输出，至少8条：
- 特征F1：...
- 特征F2：...

8、术语解释
- 对核心术语给规范定义

9、权利要求布局建议
9.1 独立权利要求：方法、系统、设备、存储介质
9.2 从属权利要求：参数范围、约束条件、优化策略、兼容性

10、审查风险与规避建议
- 新颖性风险点 + 补强建议
- 创造性风险点 + 补强建议
- 公开充分风险点 + 补强建议

【附加输出】
A. 给出“可直接写入权利要求1”的方法步骤版本（A1/A2/A3...）。
B. 给出“最容易被审查员质疑的3个点”及应对话术。
C. 给出“代理师二次加工清单”（还需补充的数据、图、参数区间）。

【输入材料】
<<<在这里粘贴论文摘要、方法章节、图示说明、实验结论、关键参数、系统架构描述>>>
```

---

## 用法建议
1. 先用原始论文材料跑一版。
2. 再补“参数范围 + 实施例细节 + 对比实验点”跑第二版。
3. 最后让模型单独输出“权利要求书草案（方法/系统）”。
